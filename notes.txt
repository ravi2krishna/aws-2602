        -> IaaS (Infrastructure as a Service)

            -> Provides raw computing infrastructure virtual machines, servers, storage, and networking. 
            You manage the operating system and applications.

            -> https://miro.medium.com/v2/resize:fit:1400/1*VIzchB0D3t1awTzj3sfe1g.png

            -> Server For Myntra Application 

                -> Hardware will be provided and managed by vendor 

                    -> 4 CPU's - 16 GB RAM - 500 GB SSD - Networking 
                
                -> Customer(You) need to manage all below 
                    
                    -> Linux Ubuntu Operating System > Tomcat > Java Runtime > Database Server (MySQL) > Myntra App 

                -> Examples: AWS EC2 - AWS VPC - AWS EBS 


        -> PaaS (Platform as a Service)     

            -> Delivers a framework for "developers" to build, test, and 
            deploy software without managing the underlying hardware.

            -> Server For Myntra Application 

                -> Hardware & Platform will be provided and managed by vendor 

                        -> 4 CPU's - 16 GB RAM - 500 GB SSD - Networking - Linux Ubuntu Operating System > Tomcat > Java Runtime > Database Server (MySQL)
                    
                    -> Customer(You) need to manage all below 
                        
                        -> Myntra App 

                    -> Examples: AWS Elastic Beanstalk - AWS RDS - AWS EKS 


        -> SaaS (Software as a Service)

            ->  Fully functional, ready-to-use software applications 
            accessed via a web browser (e.g., Salesforce, Microsoft 365, Google Workspace).
                    
            -> Server For Myntra Application 

                -> Hardware & Platform & Software will be provided and managed by vendor 

                        -> 4 CPU's - 16 GB RAM - 500 GB SSD - Networking - Linux Ubuntu Operating System > Tomcat > Java Runtime > Database Server (MySQL) > Your Application 
                    
                    -> Customer(You) need to manage all below 
                        
                        -> NOTHING 

                    -> Examples: AWS Chime(Online Meetings, Video Conference) - 
                    AWS Workspaces(Desktop As Service) - AWS Connect(Contact Center Service)

        
        -> Customer-1: Needs Only Hardware i.e Only IaaS

        -> Customer-2: Needs Hardware + Platform i.e IaaS + PaaS 

        -> Customer-3: Needs Direct Application i.e SaaS 


    -> IaaS: Rent Infrastructure - You Manage OS, Middleware & Apps - AWS EC2, AWS VPC, AWS EBS

    -> PaaS: Build & Deploy Apps - You Manage Apps - AWS Elastic Beanstalk - AWS RDS, AWS EKS 

    -> SaaS: Use Apps - vendor manages everything - AWS Chime - AWS Connect - AWS Workspaces

        -> In terms of AWS, infrastructure is made up of

            -> Regions are designed to service AWS customers (or your users) 
            that are located closest to a region.

        -> Availability Zones (AZ)

            -> Availability zone is where the actual data centers are located

            -> Within a Region, there can be multiple Availability zones

            -> Each AZ has independent Power Supply, Networking, Cooling System, Physical Security.

            -> https://www.researchgate.net/profile/Ivan_Beres/publication/360541079/figure/fig1/AS:1154856056291328@1652350548135/AWS-Regions-and-Availability-Zones-Scott-nd.ppm


        -> Edge Locations 

            -> NOTE: To understand about Edge Locations, we need to know about 
            CDN - Content Delivery Network 

            -> CDN - A Content Delivery Network (CDN) is a geographically distributed 
            group of servers that speeds up internet content delivery by 
            caching assets closer to users. 
            By reducing the physical distance data must travel, CDNs minimize latency.

            -> https://www.muvi.com/wp-content/uploads/2017/07/CDN_Muvi.png

            -> In terms of AWS Edge Server is Edge Location 

                -> 750+ CloudFront POPs and 15 Regional edge caches

            
            -> An AWS Edge Location is a data center in the AWS global network used by services 
            like CloudFront, Global Accelerator, and Lambda@Edge to store cached content 
            (images, videos) and process requests closer to end-users, 
            drastically reducing latency by serving data from the nearest point of presence(pop) 
            rather than a distant main AWS Region. 
            They act as caching warehouses, improving performance for websites.

    -> VPC (Virtual Private Cloud)

        -> A Virtual Private Cloud (VPC) is a logically isolated, private network environment 
        within a public cloud. VPC gives organizations total control over networking components
        like IP addresses, subnets, route tables, and gateways—while benefiting from the 
        scalability and cost-efficiency of shared public cloud infrastructure.

        -> https://www.akamai.com/site/en/images/article/2023/akamai-public-private-cloud.png

        -> https://images.netcomlearning.com/cms/images/virtual-private-cloud.jpg

        -> NOTE: Having VPC is similar to having your own data center inside AWS. 

        -> VPC is a service that lets customers(IBM) provision a logically isolated 
        section of the AWS Cloud where you can launch AWS resources(Servers) 
        in a VIRTUAL NETWORK that you define.

        -> Most Popular Applications
        
            -> https://en.wikipedia.org/wiki/Programming_languages_used_in_most_popular_websites

        -> VPC Architecture Reference 

            -> https://media.licdn.com/dms/image/v2/D4D12AQFI8sqvP4xFFQ/article-cover_image-shrink_720_1280/article-cover_image-shrink_720_1280/0/1700537758681?e=2147483647&v=beta&t=zE97TFGw5FQJEjUksqx7SohNsqqV57-IYQ_pj7K2M2U

        -> NOTE: VPC forms the backbone for Infrastructure to build any Application on cloud 

    
    -> VPC Components

        -> Subnets
        
            -> Segments of a VPC's IP address range that group resources based on 
            security and connectivity needs (e.g., public subnets for websites, 
            private subnets for databases).

            -> A subnet or sub network is an logical subdivision of an Amazon Virtual Private Cloud (VPC)

        -> Route Tables
        
            -> Rules that dictate where network traffic is directed.

            -> A route table contains a set of rules called routes, 
            that are used to determine where network traffic is directed.

        -> Gateways
        
            -> Connection points that bridge your VPC to the external internet or 
            on-premises data centers.

            -> Internet Gateway

                -> An Internet Gateway must be attached to a VPC if the resources inside the VPC 
                need to connect to resources via the open internet.

        
        -> Network ACLs (NACL)
        
            -> Virtual firewalls that control inbound and outbound traffic at the subnet level.

        -> Security Groups  
        
            -> Virtual firewalls that control inbound and outbound traffic at the instance level.
